프로젝트 개요

RunBoo는 런닝을 단순한 운동이 아닌 게임처럼 즐길 수 있도록 설계한 런닝 앱입니다. 5km·10km 기록을 기반으로 티어를 부여하고, 도전과제 클리어를 통해 뱃지를 획득하는 게임화(Gamification) 요소를 통해 런닝 지속성을 높이는 것이 핵심 목표입니다. 고스트 모드, 주변 러너 탐색, AI 분석 리포트 등의 기능도 제공합니다.

티어 시스템 — 백엔드

5km · 10km 런닝 기록을 기반으로 사용자에게 티어를 부여하는 시스템을 설계하고 구현했습니다. 거리 종류(DistanceType)별로 티어를 독립적으로 관리하며, 기록이 이미 있으면 업데이트하고 없으면 새로 저장하는 Upsert 패턴을 단일 메서드 안에 담았습니다.

API Endpoints

POST
/api/user-tier/save

런닝 완료 후 tierId · distanceType을 받아 티어를 저장/갱신. 동일 거리 타입의 기록이 있으면 UPDATE, 없으면 INSERT (Upsert)

GET
/api/user-tier/

인증된 사용자의 모든 거리 타입별 티어 ID 목록 반환. 프론트에서 현재 티어 뱃지 렌더링에 사용

핵심 구현 포인트

DistanceType Enum

KM_5 / KM_10 두 타입으로 티어를 분리 관리. JSON 직렬화 시 @JsonProperty("5k")로 프론트와 값 매핑

JPA Dirty Checking Upsert

@Transactional 내에서 기존 엔티티를 조회 후 updateTier() 호출 시 별도 save 없이 변경 감지로 자동 UPDATE

Optional 기반 분기

findByUserAndDistanceType()의 결과가 isPresent()이면 update, 아니면 신규 생성·저장

Stream으로 ID 추출

사용자 티어 목록에서 stream().map(ut → ut.getTier().getTierId())로 ID만 추출해 경량 응답 반환

도전과제 시스템 — 백엔드

시즌 단위로 동작하는 단계적 도전과제 시스템을 설계하고 구현했습니다. 레벨 1~30까지 순차적으로 잠금 해제되는 구조로, 완료 시 뱃지 보상 지급 + 푸시 알림 발송까지 한 트랜잭션 안에 처리됩니다.

LOCKED → IN_PROGRESS → COMPLETED

API Endpoints

POST
/api/challenges/initialize

신규 가입 또는 새 시즌 진입 시 호출. 현재 시즌의 레벨 1~3을 자동 생성하고 레벨 1만 IN_PROGRESS로 활성화. 이미 참여 중이면 멱등성 보장(early return)

GET
/api/challenges/active

현재 진행 중 1개 + 다음 대기 중 2개를 묶어서 반환. 뱃지 정보(id·name·iconUrl)를 DTO에 포함해 프론트에서 별도 요청 없이 바로 렌더링 가능

PATCH
/api/challenges/progress

런닝 데이터(type, value)를 받아 진행도를 누적. 목표값 도달 시 자동으로 COMPLETED 처리 → 뱃지 지급 → 다음 레벨 활성화 → 알림 발송까지 연쇄 실행. 응답으로 최신 챌린지 상태 3개를 즉시 반환

GET
/api/challenges/completed

완료된 전체 도전과제를 완료 시간 최신순으로 조회. Join Fetch로 N+1 문제 방지

핵심 구현 포인트

멱등성 보장 (initialize)

이미 시즌에 참여 중인 유저가 다시 호출해도 중복 생성 없이 early return. existsByUserIdAndChallenge_Season()으로 사전 체크

addProgress — 오버플로 방지

Math.min(progressValue + value, targetValue)로 진행도가 목표를 초과하지 않게 cap 처리 후 boolean 반환으로 완료 여부 전달

미래 레벨 지연 생성

완료 시점에 ensureFutureLevelExists(level+3)을 호출해 다다음 레벨을 미리 LOCKED 상태로 생성. 30레벨 상한 처리 포함

뱃지 중복 지급 방지

existsByUserIdAndBadgeId()로 이미 보유한 뱃지는 재지급하지 않음. 챌린지에 뱃지가 없는 경우도 null 체크로 안전 처리

KST 시간대 처리

LocalDateTime.now(ZoneId.of("Asia/Seoul"))에 truncatedTo(ChronoUnit.SECONDS)로 나노초 제거 후 저장

완료 후 즉시 응답

PATCH /progress 호출 한 번으로 진행도 업데이트 → 완료 처리 → 다음 레벨 오픈까지 완료하고 최신 챌린지 상태 3개를 바로 반환해 프론트 추가 요청 최소화

프론트엔드 연동 — React + TypeScript

백엔드 API와 React(TypeScript) 프론트엔드를 연동했습니다. 중앙 디자인 시스템(theme.ts)을 기반으로 컴포넌트별 스타일 파일에서 getStyles(colorScheme) 패턴으로 티어 및 도전과제 UI를 구현하고 API와 연결했습니다.

API 호출 연동

티어 저장·조회, 도전과제 초기화·진행도 업데이트·조회 API를 프론트 화면과 연결

테마 시스템 적용

useColorScheme() 훅으로 라이트/다크 모드 자동 감지 후 동적 스타일 적용

컴포넌트 분리

[Component].tsx + [Component].styles.ts 구조로 관심사 분리
